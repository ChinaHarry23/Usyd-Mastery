Unit goal. COMP9123 is your formal introduction to algorithms and data structures so you can design your own structures and algorithms and be confident of their correctness and efficiency.

The course also aims to build an analytical mindset: for a real problem, you can choose suitable structures and algorithms, implement them, and test solutions with confidence.

What is an algorithm?

An algorithm is a step-by-step procedure (a finite sequence of well-defined rules) to solve a problem or perform a task. The slides highlight three characteristics:

  • Finite — there is a clear beginning and end.
  • Precise — each step is unambiguous.
  • Effective — it produces the intended result for valid inputs.

Algorithms appear everywhere (navigation, scheduling, search). The word algorithm traces to the 9th-century scholar Al-Khwārizmī (Latin Algoritmi), whose work on systematic procedures helped shape algebra and computation.

Three abstractions used in this unit

  1. Computational problem — specifies the task: what the input is and what the output should be (the specification).
  2. Algorithm — a step-by-step method that maps inputs to outputs. This is not the same as source code: the same algorithm can be implemented in many languages.
  3. Correctness & complexity analysis — a proof or argument that the algorithm solves the problem, plus bounds on time and space (e.g. big-O) as the input size grows.

Worked pattern: “largest of three numbers”

The slides compare plain-English steps with pseudocode: compare pairs, track the current champion, then compare with the third value. For \(n\) numbers, the natural extension initialises largest ← numbers[1] and scans \(i = 2..n\) — the same logical structure as the three-number version.

Data structures (preview)

A data structure organises data so it can be accessed and updated efficiently — often to support a particular algorithm. Week 1 introduces:

  • Arrays — fixed-size contiguous storage; get/set by index in \(O(1)\) in the usual RAM model.
  • Lists (Python list) — dynamic, ordered; still \(O(1)\) index access on average in CPython’s list implementation for many operations.
  • Trees — hierarchical nodes (e.g. binary tree).
  • Graphs — vertices and edges; often stored as an adjacency list mapping each vertex to its neighbours (as in the lecture’s Python-style example).
