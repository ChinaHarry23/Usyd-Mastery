Lecture 1 includes a compact Python refresher: types, control flow, functions, and classes. Later weeks use Python for reference implementations of data structures.

Core types (high level)

Python distinguishes several built-in categories — text (str), numeric (int, float, complex), sequences (list, tuple, range), mappings (dict), sets (set, frozenset), Booleans, binary types, and None. For this unit you will mostly use int, list, tuple, dict, and small classes you define yourself.

Collections you will see immediately

  • List — ordered, mutable: my_list = [1, 2, 3, 4].
  • Tuple — ordered, immutable: my_tuple = (1, 2, 3).
  • Dictionary — keys to values: {"name": "Alice", "age": 25}.

Control flow

  • Branching — if / elif / else.
  • Iteration — for i in range(n): and while condition:.

Functions & classes

Define reusable logic with def name(args): and return. A class is a blueprint for objects (attributes and methods). In real Python the keyword is class (lowercase); the lecture slide’s Class capitalisation is a typo.

Arrays vs Python lists

Many languages have fixed-length arrays with \(O(1)\) indexing; Python’s list is a dynamic array abstraction — still \(O(1)\) random access in the algorithm-analysis sense for index operations, with resizing handled for you. When analysing pseudocode on a fixed array A[1..n], think “indexed sequence” whether the source is a C array or a Python list.
